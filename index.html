Presentación Educativa

¿Por qué existen? (Motivación)

Las listas ligadas simples solo permiten el recorrido en una sola dirección. Esto genera ineficiencias cuando:

  • Necesitamos acceder al nodo anterior frecuentemente.
  • Queremos eliminar un nodo dado sin tener que recorrer toda la lista.
  • Necesitamos recorrer la estructura en reversa.

Solución: Agregar un puntero adicional que apunte al nodo previo.

Listas Doblemente Ligadas

Cada nodo contiene tres campos:

  1. Puntero al nodo Anterior (prev).
  2. El Dato (data).
  3. Puntero al nodo Siguiente (next).

Ejemplos en la Industria

¿Cuándo se usan las listas doblemente ligadas en el mundo real?

  • Navegadores Web: El historial de páginas "Atrás" y "Adelante". Cada página es un nodo, puedes retroceder (prev) o avanzar (next) en O(1).
  • Editores de Texto/Software: La funcionalidad de "Deshacer" (Undo) y "Rehacer" (Redo). Mantienes el estado actual y puedes moverte libremente entre estados pasados y futuros.
  • Sistemas Operativos: Gestión de hilos (threads) y procesos en ejecución en las colas del scheduler, donde necesitan ser removidos o insertados rápidamente.

Ventajas vs Desventajas

✅ Ventajas ❌ Desventajas
Recorrido bidireccional fluido. Requieren más memoria (un puntero extra).
Eliminación en O(1) si se tiene el nodo. Lógica de inserción/eliminación más extensa.

Código: Estructura del Nodo

class NodoDoble<T> {
    public T dato;
    public NodoDoble<T> prev; // Puntero al anterior
    public NodoDoble<T> next; // Puntero al siguiente

    public NodoDoble(T dato) {
        this.dato = dato;
        this.prev = null;
        this.next = null;
    }
}

Ejemplos en la Industria (Circulares)

  • Reproductores de Música: Playlists cíclicos. Cuando llegas a la última canción y presionas "Siguiente", pasas a la primera canción.
  • Sistemas Operativos (Alt-Tab): El carrusel para cambiar entre aplicaciones abiertas. Presionar Alt+Tab avanza continuamente en ciclo.
  • Juegos Multijugador por Turnos: Mantener la estructura de turnos. Al terminar el turno del último, vuelve automáticamente al primero.

Código: Inserción Circular Doble

public void insertar(T dato) {
    NodoDoble<T> nuevo = new NodoDoble<>(dato);
    if (cabeza == null) {
        cabeza = nuevo;
        cabeza.next = cabeza; // Se apunta a sí mismo
        cabeza.prev = cabeza;
    } else {
        NodoDoble<T> ultimo = cabeza.prev; // O(1)
        
        nuevo.next = cabeza;
        nuevo.prev = ultimo;
        ultimo.next = nuevo;
        cabeza.prev = nuevo;
    }
}

Código: Recorrido Seguro Circular

Se debe tener cuidado de no hacer un ciclo infinito.

public void recorrerAdelante() {
    if (cabeza == null) return;
    
    NodoDoble<T> actual = cabeza;
    do {
        System.out.print(actual.dato + " <-> ");
        actual = actual.next;
    } while (actual != cabeza); // Condición de parada vital
    
    System.out.println("(Vuelta al inicio)");
}

Complejidad (Big-O)

Análisis temporal y espacial de las operaciones.

Operación Lista Doble Circular Doble Razón
Acceso / Búsqueda O(n) O(n) No tienen un índice directo; hay que recorrer nodo a nodo.
Inserción (Extremos) O(1) O(1) Si hay puntero a cola (Doble) o con cabeza.prev (Circular), es instantáneo.
Inserción (En Medio) O(n) O(n) Toma O(n) buscar la posición, y O(1) desligar y conectar los punteros.
Eliminación (Extremos) O(1) O(1) Muy rápido gracias a los punteros extremos prev y next.
Eliminación (En Medio) O(n) O(n) Se repite el patrón de la inserción: buscar cuesta O(n), borrar cuesta O(1).
Memoria Espacial: Ambas consumen O(n) de memoria total. Tienen una pequeña sobrecarga por usar más bytes: cada nodo almacena dos referencias (next y prev), consumiendo más RAM que un arreglo tradicional o una lista simple.

Errores Típicos de Programación

  • NullPointerException (Listas Dobles): Ocurre al intentar acceder a nodo.prev cuando nodo es la cabeza, o a nodo.next cuando es la cola.
  • Bucles Infinitos (Circulares): Al recorrer una lista circular con un while(actual != null) la lista nunca terminará. Siempre se debe usar una condición de parada como while(actual != cabeza).
  • Perder la Referencia: Al insertar o eliminar, si no se actualizan los 4 punteros implicados (el next y prev de ambos nodos vecinos), la cadena se rompe y se pierde memoria.
  • No actualizar la cabeza/cola: Olvidar reasignar cabeza = nuevoNodo si la inserción ocurre al principio de la lista.

Casos Extremos (Eliminar)

Siempre debes validar estos 4 escenarios críticos al intentar borrar un nodo:

Caso Crítico Problema y Solución
Lista Vacía Intentar borrar con cabeza == null lanza una excepción. Solución: Inicia el método con if (cabeza == null) return false;
Un Solo Nodo Al borrar el único nodo, la lista queda sucia en memoria si no se limpia. Solución: La cabeza y cola deben apuntar a null.
Eliminar Cabeza Borrar el primer nodo requiere mover la cabeza. Solución: Asigna cabeza = cabeza.next y luego cabeza.prev = null (para evitar enlaces rotos).
Eliminar Cola Borrar el último nodo requiere mover la cola. Solución: Asigna cola = cola.prev y asegúrate de hacer cola.next = null.

¿Por qué NO usar Listas Dobles?

No siempre son la mejor opción. Aquí te mostramos cuándo evitarlas:

Escenario ¿Usar Lista Doble? Mejor Alternativa
Solo necesitas insertar al final y leer desde el principio. NO Lista Simple (ahorras un 50% de memoria en punteros).
Necesitas acceso inmediato por índice a cualquier elemento. NO Arrays / ArrayList (tienen acceso O(1)).
Sistemas embebidos con memoria RAM muy limitada (IoT). NO Arreglos estáticos o Listas Simples (para evitar el overhead de memoria).
Se requiere retroceder constantemente o deshacer acciones. SÍ Lista Doblemente Ligada.

👨‍💻 Ejercicio Práctico (Clase)

Reto: "El Navegador Web"

Estás construyendo la funcionalidad de historial de un navegador. Diseña un programa usando una Lista Doblemente Ligada genérica (String) que simule las siguientes acciones:

  • Visitar página (Insertar): Agrega un nuevo nodo al final y lo establece como la página actual.
  • Botón "Atrás": Se mueve al nodo prev y muestra la URL actual (Validando si existe un anterior).
  • Botón "Adelante": Se mueve al nodo next y muestra la URL actual (Validando si existe un siguiente).
  • Mostrar Historial: Muestra la ruta completa recorrida.

Conclusiones

Tipo de Lista Recorrido Acceso a Extremos Aplicación Ideal
Simple Adelante O(1) Inicio, O(n) Fin Pilas, implementaciones básicas.
Doble Bidireccional O(1) Inicio y Fin Historiales (Atrás/Adelante).
Circular Simple Continuo Adelante O(1) Fin Turnos fijos de un solo sentido.
Circular Doble Continuo Bidireccional O(1) Inicio y Fin Carruseles, Playlists cíclicos.
